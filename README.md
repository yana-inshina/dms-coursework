# Информационно-справочная система медицинского страхования (ДМС)

Курсовая работа по направлению  
«Прикладные информационные системы в экономике и финансах».

---

## Назначение системы
Информационно-справочная система медицинского страхования предназначена
для автоматизации процессов работы со страховыми программами ДМС,
индивидуальными и корпоративными заявками, а также страховыми полисами.

Система ориентирована на использование в страховой компании и
обеспечивает централизованное хранение данных, автоматизированный расчет
страховой премии и удобный веб-интерфейс для различных категорий
пользователей.

---

## Основной функционал

### Для индивидуальных клиентов
- просмотр каталога страховых программ ДМС;
- расчет страховой премии по выбранной программе;
- подача онлайн-заявки на страхование;
- отслеживание статуса заявки;
- получение электронного страхового полиса.

### Для корпоративных клиентов
- создание карточки организации;
- подача заявок на коллективное страхование сотрудников;
- расчет страховой премии с учетом количества сотрудников и параметров группы;
- просмотр статусов корпоративных заявок и оформленных полисов.

### Для менеджеров страховой компании
- регистрация новых клиентов (индивидуальных и корпоративных);
- просмотр и обработка заявок клиентов;
- одобрение или отклонение заявок с комментариями;
- выпуск страховых полисов на основании заявок;
- оформление страхового полиса вручную (вне заявки);
- управление справочниками (программы ДМС, покрытия, исключения, акции,
  регионы, медицинские организации);
- просмотр статистики работы системы.

### Для администратора
- управление учетными записями пользователей;
- назначение и изменение ролей;
- контроль данных системы;
- просмотр статистики в режиме «только чтение».

---

## Архитектура системы
Система реализована по клиент-серверной архитектуре с использованием
шаблона MVC (Model–View–Controller).

- клиентская часть реализована в виде веб-интерфейса;
- серверная часть обрабатывает бизнес-логику и запросы;
- данные хранятся в реляционной базе данных MySQL.

Для обеспечения безопасности используется ролевая модель доступа (RBAC),
реализованная средствами Spring Security.

## Структура проекта
- `src/main/java` — серверная часть приложения (контроллеры, сервисы,
  модели, репозитории);
- `src/main/resources` — конфигурационные файлы и шаблоны представления;
- `pom.xml` — конфигурация Maven и список зависимостей.

---

## Запуск проекта
1. Установить JDK 17 и СУБД MySQL.
2. Создать базу данных и указать параметры подключения в файле
   `application.properties`.
3. Запустить приложение через главный класс  
   `DmsIsmsApplication`.
4. Открыть в браузере адрес:  
   `http://localhost:8080`.
